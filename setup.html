Before you start

GameNative build note

If custom games or the expected drive do not appear, re-check the GameNative build you installed before changing random settings. The home page explains the modern prerelease limitation.

New to these terms?

Use the Glossary when a term is unclear. Do not guess at settings like DXVK, FEXCore, Turnip, present mode, shared storage, or plugin order.

How to troubleshoot setup

  • Change one thing at a time, then test again.
  • Write down exact version numbers before changing GameNative, Proton/Wine, FEXCore, DXVK, Turnip, DML, or driver settings.
  • Test vanilla before adding mods. A modded crash report is not useful if the unmodded game never launched.
  • Keep screenshots of GameNative settings, DML profile settings, deployment results, and plugin output.
  • Separate the problem into one layer: GameNative, the game install, DML, the mod list, the load order, or INI settings.

Proton 11 ARM64EC warning

Use Proton 10 ARM64EC for the normal guide target. GameNative Proton Wine releases currently mark Proton 11 ARM64EC builds as testing-only and warn about instability or crashes. Test Proton 11 only when you are deliberately comparing Proton versions.

Device and version profile

Pick the device/profile closest to your setup. The values below are test notes for GameNative and the compatibility layers, not a guarantee that every matching device will work.

GOG launcher notice inside GameNative

If the Fallout: New Vegas launcher inside the GameNative Windows container has greyed-out options, hides the Play button, or says the game needs to be installed even though the files are already on the device, the container may be missing the Bethesda registry entry.

  1. Download GOG's beth_registry.zip fix.
  2. Extract beth_registry.cmd.
  3. Place beth_registry.cmd in the game root inside the container, next to FalloutNV.exe.
  4. Run it from that folder and allow it to update the registry.
  5. Open the launcher again. If the options are still greyed out, report the GameNative build, game source, and exact folder path.

This is not a normal PC setup step. Use it only when the game is already on the Android device and the launcher inside GameNative acts like the game is not installed.

Checkpoint: GameNative works before DML

  • The game launches in GameNative before mods are installed.
  • You can reach the main menu.
  • You can start a new game and reach Doc Mitchell's house.
  • If this fails, stop here. Fix GameNative, the game install, or the profile before adding mods.

DML setup

Create a fresh profile in Droid Mod Loader for New Vegas.

Install New Vegas where DML can reach it

Fallout: New Vegas needs to be installed outside GameNative App Storage. Put the game in shared storage so Android and DML can access the folder through the system file picker.

/storage/emulated/0/PcGames/Fallout New Vegas/

If the game lives inside GameNative's private App Storage, DML will not be able to use the real Data folder as a deployment target.

  1. Install or move Fallout: New Vegas to shared storage, not GameNative App Storage.
  2. Open Droid Mod Loader.
  3. Create or select a Fallout: New Vegas profile.
  4. Name it Pocket Mojave.
  5. Set the deployment target to the game's Data folder in shared storage.
  6. Confirm that the target contains FalloutNV.esm.
  7. Do not select the game root as the Data target.
  8. Install mods in the order shown in the mod list.

Plugin order and mod priority are not the same thing

Plugin order controls ESM and ESP loading. Mod priority controls loose file conflicts. DML has to handle both, but they are different lists.

Checkpoint: DML deployment target

  • DML shows a Pocket Mojave profile.
  • The deployment target is the game Data folder in shared storage.
  • The target folder contains FalloutNV.esm.
  • If DML points at the game root or GameNative App Storage, stop and fix the target before installing mods.

BethINI Pie

Use BethINI Pie to create simple, predictable INI settings before you start piling on mods.

  1. Open BethINI Pie in the same Windows/GameNative environment that can see your New Vegas INI files.
  2. Select Fallout: New Vegas.
  3. Point it to the active GameNative/New Vegas profile if it does not detect the path automatically.
  4. Open BethPresets.
  5. Choose Medium.
  6. Set texture quality to the highest/max option.
  7. Keep resolution at 1280x720 for the first guide pass.
  8. Enable audio multithreading.
  9. Save the INIs and launch the game once before installing more mods.

Audio cache caution

Audio multithreading is recommended. Audio cache tweaks can be unstable in Windows containers, so only change audio cache settings if you are deliberately testing audio problems.

Tracking performance

  1. Use the GameNative overlay first for a quick FPS check.
  2. Use the same route, weather, time of day, and save each time.
  3. Only switch to DXVK HUD when you need frame time, draw call, pipeline, or memory details.
  4. Turn the DXVK HUD off after the benchmark.
DXVK_HUD=fps,frametimes,drawcalls,pipelines,memory
