Before you start

GameNative build note

If custom games or the expected drive do not appear, re-check the GameNative build you installed before changing random settings. The home page explains the modern prerelease limitation.

New to these terms?

Use the Glossary when a term is unclear. Do not guess at settings like DXVK, FEXCore, Turnip, present mode, shared storage, or plugin order.

Which PC version should I use?

GOG is recommended, but Steam and/or Epic Games versions should be usable

The GOG Ultimate Edition is the simplest choice for this guide because it is DRM-free and is generally easier to manage as a custom game in GameNative.

Steam and Epic Games versions should also work. GameNative can recognize Steam ownership, but a New Vegas folder copied manually from a PC may not be detected immediately.

If your Steam installation is not recognized in GameNative, see the "How to troubleshoot setup" section below.

These DML instructions are for v0.6.0-beta

DML v0.6.0-beta uses separate manually selected targets for the Data folder and, when needed, the Game Root folder. It does not infer one from the other or validate the complete game installation. Plugin files still require a manual handoff to GameNative. This release does not create DML_output, apply INI presets, or manage archive invalidation.

Installation instructions

In order for Droid Mod Loader to be able to modify game files, Fallout: New Vegas must be installed in Android Shared Storage.

This means that you should not use GameNative's built-in installer in order to populate game files, as this will put the game files into Application Storage, which is managed by GameNative and inaccessible by Droid Mod Loader.

Instead, from your PC, perform a clean install of Fallout: New Vegas. The Installation Guide at "The Best of Times" is an excellent resource for getting a clean install. Once you've reached the "Post Installation" stage in that guide, you should have a proper clean installation of the game.

Next, move the complete installation folder from your PC over to your Android device into shared storage. In our example, we will move it into Games/FalloutNV.

Launch GameNative, and add Fallout: New Vegas as a custom game by pointing GameNative to your installation folder (eg. Games/FalloutNV).

Finally, launch the game from within GameNative. If it doesn't immediately launch, don't panic! This is very common, and likely easy to fix.

How to troubleshoot setup

My game doesn't launch!

If you have just followed the installation instructions above, attempted to launch the game and just got a black screen, try the following steps:

  1. Click on the "Settings" button of your Fallout: New Vegas custom game screen in GameNative to launch the container settings.
  2. Click "Open Container".
  3. Inside of the container, it will look similar to a Windows desktop. If a file browser opens automatically, click into A:.
  4. Double-click into FalloutLauncher.exe.
  5. If the Fallout: New Vegas launcher has greyed-out options, hides the Play button, or says the game needs to be installed even though the files are already on the device, the container is likely missing the Bethesda registry entry.

In order to add the Bethesda registry entry into the container, follow these steps:

  1. Download GOG's beth_registry.zip fix.
  2. Extract beth_registry.cmd.
  3. Place beth_registry.cmd in the game root inside the container, next to FalloutNV.exe.
  4. Run it from that folder and allow it to update the registry.
  5. Open the launcher again. If the options are still greyed out, report the GameNative build, game source, and exact folder path.

General Troubleshooting Steps

  • Change one thing at a time, then test again.
  • Write down exact version numbers before changing GameNative, Proton/Wine, FEXCore, DXVK, Turnip, DML, or driver settings.
  • Test vanilla before adding mods. A modded crash report is not useful if the unmodded game never launched.
  • When DML asks for the Data Folder, select <Game Folder>/Data. Confirm it contains FalloutNV.esm.
  • When DML separately asks for the Game Root Folder, select the parent folder containing FalloutNV.exe and Data. Do not swap the two targets.
  • If DML rejects a folder or deploys to the wrong place, reselect the correct target and manually check its contents. v0.6.0-beta does not validate the complete New Vegas installation for you.
  • If deployment worked before but now fails, Android may have removed DML's folder permission. Select the affected folder again and approve the system access prompt.
  • A missing DML_output folder is normal in v0.6.0-beta. That automatic export folder is planned for a later release.
  • Keep screenshots of GameNative settings, DML profile settings, deployment results, the plugin list, and DML's operation log.
  • Separate the problem into one layer: GameNative, the game install, DML, the mod list, plugin activation, timestamp order, or INI settings.

Troubleshooting Steam/Epic Games version installation

If GameNative does not recognize a copied Steam installation:

  1. Confirm that GameNative is signed into the Steam account that owns Fallout: New Vegas.
  2. Rename the shared-storage game folder so it closely matches the name shown in the Steam client. Fallout New Vegas is a good example.
  3. Close and reopen GameNative.
  4. Refresh or recreate the game entry or container, then select FalloutNV.exe again.

The folder rename and container refresh worked during my testing, but newer GameNative versions may not need it and the workaround is not guaranteed on every device.

The Epic Games Store version should also work with the appropriate Epic preparation steps, but I have not personally tested it. GOG remains the recommended first-time route.

DML v0.6.0-beta setup

Create a fresh profile in Droid Mod Loader for New Vegas.

Install New Vegas where DML can reach it

Fallout: New Vegas needs to be installed outside GameNative App Storage. Put the game in shared storage so Android and DML can access the folder through the system file picker.

/storage/emulated/0/PcGames/Fallout New Vegas/

If the game lives only inside GameNative's private App Storage, DML cannot use the real Data folder as a deployment target.

  1. Install or move Fallout: New Vegas to shared storage, not GameNative App Storage.
  2. Open Droid Mod Loader.
  3. Create or select a Fallout: New Vegas profile and name it Pocket Mojave.
  4. When DML asks for the Data Folder, select <Game Folder>/Data. Confirm that it contains FalloutNV.esm.
  5. If DML separately asks for the Game Root Folder, select the parent folder containing FalloutNV.exe and the Data folder.
  6. Do not swap those two targets. v0.6.0-beta does not automatically detect Data from the Game Root or validate the complete installation.
  7. Install and deploy mods in the order shown in the mod list.
  8. Open DML's plugin list, enable only the plugins you installed, arrange the list to match the guide, and tap Write Plugin Files.

Activation, plugin order, and mod priority are different

plugins.txt lists enabled plugins. For Fallout: New Vegas, each enabled filename is written on its own line without a * prefix. loadorder.txt records DML's full plugin list in the order shown by the app, including disabled entries.

New Vegas applies plugin order from the modification timestamps of the ESM and ESP files in Data. DML v0.6.0-beta does not change those timestamps, so writing or copying loadorder.txt alone does not enforce the real in-game order. Mod priority is a separate list that controls loose-file conflicts.

Checkpoint: DML deployment target

  • DML shows a Pocket Mojave profile.
  • The Data Folder target is <Game Folder>/Data in shared storage and contains FalloutNV.esm.
  • If a Game Root target is configured, it is the parent folder containing FalloutNV.exe and Data.
  • Neither target points into locked GameNative App Storage.
  • DML reports that plugins.txt and loadorder.txt were written successfully.

Copy DML's plugin files into GameNative

DML v0.6.0-beta does not copy these files into GameNative automatically.

  1. After tapping Write Plugin Files, open DML's operation log. It prints the exact source path for plugins.txt and loadorder.txt.
  2. On most current installations, the files are under DML's profile storage in Android/data/com.shonkware.droidmodloader/files/state/profiles/. Open the folder for the active Pocket Mojave profile. If DML's log shows a different path, use the logged path.
  3. Use ZArchiver or another file manager that can request access to Android/data through Android's system folder picker. Root access is not normally required for this external app-storage path.
  4. Copy both files into a temporary shared folder that GameNative can see, such as Download/DML Plugin Transfer/. This folder is user-created; it is not the unreleased automatic DML_output feature.
  5. Open GameNative's file manager and copy both files into C:/users/xuser/AppData/Local/FalloutNV/. If your container uses another username, use that user's AppData/Local/FalloutNV folder.
  6. Replace the existing files when prompted. plugins.txt controls activation; loadorder.txt is a readable reference for DML's saved list. Repeat this handoff whenever plugin activation or the DML list changes.

Checkpoint: GameNative sees the files

  • plugins.txt and loadorder.txt exist in GameNative's AppData/Local/FalloutNV folder.
  • plugins.txt contains only the plugins you enabled, with no * prefixes.
  • The ESM and ESP timestamps in the actual shared-storage Data folder used by GameNative match the intended New Vegas order.

BethINI Pie

Use BethINI Pie to create simple, predictable INI settings before you start piling on mods.

  1. Open BethINI Pie in the same Windows/GameNative environment that can see your New Vegas INI files.
  2. Select Fallout: New Vegas.
  3. Point it to the active GameNative/New Vegas profile if it does not detect the path automatically.
  4. Open BethPresets.
  5. Choose Medium.
  6. Set texture quality to the highest/max option.
  7. Keep resolution at 1280x720 for the first guide pass.
  8. Enable audio multithreading.
  9. Save the INIs and launch the game once before installing more mods.

Audio cache caution

Audio multithreading is recommended. Audio cache tweaks can be unstable in Windows containers, so only change audio cache settings if you are deliberately testing audio problems.

Tracking performance

  1. Use the GameNative overlay first for a quick FPS check.
  2. Use the same route, weather, time of day, and save each time.
  3. Only switch to DXVK HUD when you need frame time, draw call, pipeline, or memory details.
  4. Turn the DXVK HUD off after the benchmark.
DXVK_HUD=fps,frametimes,drawcalls,pipelines,memory
