Reports

Reports should include enough detail for someone else to reproduce the problem instead of guessing at random settings.

Submit a report

Use the Device Compatibility Submissions page for tested device results. A useful report needs device, driver, GameNative, DML, Proton/Wine, FEXCore, DXVK, Turnip, storage, and test-result details. Vague "it works" reports are not enough to list a device as compatible.

If you are reporting a repeatable problem, open an issue and include your device, GameNative version, DML version, profile JSON, screenshots, logs, and the exact step where it failed.

Open a Pocket Mojave GitHub issue

Before opening an issue

  • Confirm vanilla GameNative worked before reporting a modded crash.
  • Write down exact versions for GameNative, Proton/Wine, FEXCore, DXVK, Turnip/driver, DML, and the guide date.
  • Separate the failure point: GameNative launch, game install, DML deployment, plugin order, INI settings, or a specific mod.
  • Attach crash logs only when available. Screenshots of settings are still useful.
