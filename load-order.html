Load Order

DML handles mod priority and plugin order separately. This page is about plugin order.

Use LOOT as a helper

LOOT supports Fallout: New Vegas and can help sort plugins, but do not treat it as magic. Use it to get warnings and a suggested order, then apply the order in DML.

  1. Copy or expose your DML-installed Data folder to a PC or Windows environment where LOOT can see the plugins.
  2. Open LOOT.
  3. Select Fallout: New Vegas.
  4. Let LOOT detect the installed plugins.
  5. Sort the plugins.
  6. Read the warnings. Missing masters matter more than the final order.
  7. Write down or screenshot the sorted order.
  8. Do not blindly copy advice for mods you skipped.
  9. Apply the final order manually in DML.
Important
If LOOT says a plugin has a missing master, stop. Do not launch the game until the missing master is installed, disabled properly, or the dependent plugin is removed.

Sort plugins in Droid Mod Loader

  1. Open the Pocket Mojave profile.
  2. Deploy Current State.
  3. Run plugin discovery.
  4. Open the plugin list.
  5. Enable the plugins used by this guide.
  6. Move official masters to the top if they are not already there.
  7. Put YUP after the official DLC.
  8. Put other ESM files after YUP unless their mod page says otherwise.
  9. Put bug-fix ESP files before gameplay and patch ESP files.
  10. Put patches after the mods they patch.
  11. Export plugin outputs.
  12. Launch the game through GameNative.
Rule of thumb
Masters first, fixes early, UI/gameplay after fixes, patches last.

Where the plugin files go

The usual Windows paths are:

%LOCALAPPDATA%/FalloutNV/plugins.txt
%LOCALAPPDATA%/FalloutNV/loadorder.txt

Inside a Proton or Wine prefix, that usually means:

drive_c/users/<user>/AppData/Local/FalloutNV/

The exact GameNative folder can vary. If the game is ignoring your load order, check this path first.
