Tools and Mods to Avoid

This page is not a permanent ban list. It is a mobile-first caution list for this guide.

Tools to avoid for this guide

ENB

Avoid

Not a good fit for this Android/GameNative target.

Wabbajack

Avoid

Desktop MO2 automation does not match the current DML mobile setup.

Automated LOD generation on Android

Avoid

Useful later, but too many generated files and too many variables for this guide right now.

Old New Vegas mod managers

Avoid

This guide is written for DML. Old managers make troubleshooting harder.

Random INI packs

Avoid

Use BethINI Pie and your own saved INIs instead.

Sound Extender NVSE

Avoid

Avoid for now. It has been associated with crashes in this Android/GameNative setup, and there is no currently known guide-safe fix.

Mod categories to avoid for now

Generated LOD setups

Avoid

Needs its own Android path later.

Huge texture packs

Testing

Can work, but storage and memory cost climb fast. Test separately.

Heavy grass or landscape overhauls

Testing

Extra render cost and more variables.

Heavy particle overhauls

Avoid

Test separately with FPS and crash logs.

Picture-in-picture scope setups

Testing

Can be expensive on mobile. Test after the main guide.

Large NPC face/body overhauls

Testing

Asset-heavy and patch-heavy.

Big quest/worldspace stacks

Testing

Add one at a time after the guide list works.

Safe modding rule

  • Do not keep adding mods after a failed launch.
  • Do not change Proton, Turnip, DXVK, BethINI, and mods all in one test.
  • Do not assume desktop advice applies cleanly to Android.
  • When benchmarking, change one thing and write down what changed.
